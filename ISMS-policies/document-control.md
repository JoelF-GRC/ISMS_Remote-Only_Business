# Document Control Policy & Register
ISO/IEC 27001:2022 Clause 7.5, Annex A: A.5.36
Last Updated: 2025-02-15

## Purpose
Define how ISMS documented information is created, reviewed, approved, versioned, distributed, and retired, satisfying Clause 7.5 (Documented information) and A.5.36 (Compliance with policies, rules and standards for information security).

## Policy

### 1. Document Creation & Approval
- All ISMS documents must include an Ownership section and, where applicable, a Document Version History table.
- New documents and major revisions require approval from the document owner and, for governance-level documents (ISMS-policies/), Leadership (CISO/CTO).

### 2. Versioning
- Documents use sequential version numbers (e.g., 1.0, 1.1, 2.0). Minor edits (typos, formatting) increment the minor version; substantive policy changes increment the major version.
- Each version change must be recorded in the document's Document Version History table with author, date, summary of changes, and approver.

### 3. Review Cycle
- All ISMS governance documents (ISMS-policies/) and IT Security policies (IT-security-policies/) must be reviewed at least annually, or sooner following significant organizational, technological, or regulatory change.
- Registers (registers/) are living documents reviewed on the cadence defined in their own ownership sections (e.g., risk register reviewed quarterly).

### 4. Storage, Access & Retention
- ISMS documents are maintained in this version-controlled repository.
- Access to edit governance documents is restricted to the Security Department; all changes are tracked via version control history.
- Superseded versions are retained in version control history rather than deleted, providing an audit trail.

### 5. Distribution
- Current versions of policies must be accessible to all relevant personnel (e.g., via the company intranet or this repository).
- Material policy changes must be communicated to affected staff.
