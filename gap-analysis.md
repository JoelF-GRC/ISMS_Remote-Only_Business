# ISO/IEC 27001:2022 Gap Analysis Tool
CompanyX ISMS
Last Updated: 2026-07-02

## Purpose
This tool helps an organization assess its current maturity against ISO/IEC 27001:2022 Clauses 4–10 and the consolidated Annex A policy areas used in this ISMS, **before** committing to a certification timeline. It is intended for use early in an implementation project — typically before the [iso27001-readiness-checklist.md](iso27001-readiness-checklist.md) is worked through in detail.

Unlike `internal-audit-checklist.md` (which assumes an operating ISMS and supports the *annual* audit cycle), this gap analysis is a **one-time or periodic baseline assessment** used to scope a certification project, build a roadmap, and prioritize effort.

## How to Use This Tool
1. For each row, assign a **Maturity Score (0–4)** reflecting the organization's current state — be honest, not aspirational.
2. Record supporting evidence or notes on the current state.
3. Identify the gap and the action required to reach Maturity 3 (Managed) as a minimum baseline for certification readiness.
4. Assign a priority, owner, and target date.
5. Use the completed table to build an implementation roadmap, starting with High-priority items scored 0–1.

## Maturity Scale

| Score | Level | Description |
|-------|-------|-------------|
| 0 | **Not Started** | No documentation, process, or awareness exists. |
| 1 | **Ad Hoc** | Informal or inconsistent practices exist but are undocumented and not repeatable. |
| 2 | **Defined** | Documented policy/procedure exists but is not consistently followed, monitored, or evidenced. |
| 3 | **Managed** | Implemented, followed, monitored, and evidenced with records. Minimum bar for certification. |
| 4 | **Optimized** | Managed, plus regularly reviewed and improved based on metrics, audits, and incidents. |
